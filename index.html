// backend/src/services/search.service.js

async function smartSearch(userId, naturalQuery) {
  // Step 1: Parse query with Claude
  const filters = await parseSearchQuery(naturalQuery);

  // Step 2: Build PostgreSQL query
  const emails = await prisma.email.findMany({
    where: {
      userId,
      AND: [
        filters.timeframe ? {
          receivedAt: { gte: filters.timeframe.start }
        } : {},
        filters.topic ? {
          OR: [
            { subject: { contains: filters.topic, mode: 'insensitive' } },
            { preview: { contains: filters.topic, mode: 'insensitive' } }
          ]
        } : {},
        filters.sender ? {
          senderEmail: { contains: filters.sender, mode: 'insensitive' }
        } : {}
      ]
    },
    include: { bucket: true },
    orderBy: { receivedAt: 'desc' }
  });

  return { emails, parsedFilters: filters };
}

  "topic": "string or null",
  "timeframe": { "start": "ISO date" } or null,

## Core Architecture

### Email Classification Flow

1. **Fetch**: Gmail API retrieves last 200 threads
2. **Parse**: Extract subject, sender, preview, timestamp
3. **Batch**: Group 20-50 emails for classification
4. **Classify**: Send batch to Claude with bucket definitions
5. **Store**: Update `bucketId` for each email in PostgreSQL

**Why batching?**
- Reduces API calls to Claude (cost optimization)
- Provides better context for classification decisions
- Manages rate limits effectively

### Smart Search Architecture

Uses a two-tier hybrid approach:

1. **Natural Language Parsing** (Claude API):
   - User query: "Find emails about the project from last week"
   - Claude extracts: topic, timeframe, sender, bucket
   - Returns structured JSON filters

2. **Database Query** (PostgreSQL):
   - Constructs Prisma query from parsed filters
   - Uses case-insensitive search on subject/preview
   - Applies date range filters for timeframe
   - Orders by relevance and recency

**Example search flow:**
```javascript
User: "emails from Sarah about budget"
  ↓
Claude parses: { topic: "budget", sender: "Sarah" }
  ↓
SQL: WHERE senderEmail ILIKE '%Sarah%' AND (subject OR preview) ILIKE '%budget%'
  ↓
Return ranked results
```

## Smart Search Implementation Strategy

### Approach: Hybrid Search (Keyword + Semantic)

**Two-tier search:**

1. **Quick Keyword Filter** (PostgreSQL `ILIKE`)
   - Search subject, sender, preview
   - Fast, handles exact matches
   - Falls back to this if LLM is slow

2. **Smart LLM Search** (Claude API)
   - Parse natural language query
   - Extract: timeframe, sender, topic, keywords
   - Generate SQL-like filter criteria
   - Optionally: semantic similarity on email content

Return ranked results
